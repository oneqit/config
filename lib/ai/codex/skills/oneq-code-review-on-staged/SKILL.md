# Staged 파일 코드 리뷰

다음 staged된 변경사항을 코드 리뷰해줘.

1. 변경 파일 목록 확인
```
git diff --staged --stat
```

2. 전체 diff 분석
```
git diff --staged
```

## 리뷰 항목

1. **보안**: 보안 취약점, 민감한 정보 노출
2. **버그 및 오류**: 잠재적인 버그, 엣지 케이스 누락, null/undefined 처리
3. **호환성/의존성**: 다른 모듈에 미치는 영향, breaking change 여부
4. **에러 핸들링**: 예외 처리 누락, 에러 메시지 적절성
5. **성능**: 불필요한 연산, 메모리 누수, N+1 쿼리 등
6. **테스트**: 테스트 누락, 기존 테스트 깨짐 가능성
7. **코드 중복**: 동일/유사 기능의 기존 함수·유틸리티가 있는데 활용하지 않고 새로 구현했는지 여부
8. **가독성**: 네이밍, 복잡한 로직, 주석 필요 여부
9. **스타일 일관성**: 주변/기존 코드의 스타일(네이밍, 포매팅, 패턴, 구조)과 유사하게 구현되었는지 여부
10. **베스트 프랙티스**: 코딩 컨벤션, 디자인 패턴

## 심각도 기준

- 🔴 **커밋 차단**: 보안 취약점, 명백한 버그, 데이터 손실·유실 가능성, breaking change 등 커밋 전 반드시 수정
- 🟡 **커밋 전 개선 권장**: 품질·유지보수성 저하, 엣지 케이스 누락, 테스트 부족 등 커밋 전 논의/개선 필요
- 🟢 **선택적 개선**: 사소한 가독성·스타일 제안, 지금 반영하지 않아도 무방
