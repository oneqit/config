# Staged 파일 코드 리뷰

다음 staged된 변경사항을 코드 리뷰해줘.
```
git diff --staged
```

## 리뷰 항목

1. **버그 및 오류**: 잠재적인 버그, 엣지 케이스 누락, null/undefined 처리
2. **보안**: 보안 취약점, 민감한 정보 노출
3. **성능**: 불필요한 연산, 메모리 누수, N+1 쿼리 등
4. **가독성**: 네이밍, 복잡한 로직, 주석 필요 여부
5. **베스트 프랙티스**: 코딩 컨벤션, 디자인 패턴

## 출력 형식

발견된 각 이슈를 심각도(🔴 높음 / 🟡 중간 / 🟢 낮음)와 함께 정리하고, 개선 코드 예시를 제안해줘.
